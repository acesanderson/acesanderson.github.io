## Content Strategy
**Source material:** Code Learnings.md in vault is the motherlode. 1+ years of dated
entries covering LLMs, embeddings, vector DBs, agents, prompt engineering, Python
patterns, home lab. Blog posts distill these into coherent, standalone pieces.

**Post types to define:**
- Deep dives: one specific build, problem, or discovery — long-form
- TILs: short, one thing learned — good for cadence
- Reviews/opinions: tech industry, online learning, tools

**Dual audience tension:** posts should be honest + personal (peer-facing) AND
coherent + polished enough to work as hiring signal. Resolution: write for peers,
don't dumb down, but give each post a clear payoff sentence.
