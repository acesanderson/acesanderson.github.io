### 1. Local Ruby vs. GitHub Actions Strategy
**Recommendation: Skip local Ruby entirely.**
Installing Ruby, `rbenv`, and managing Gems on macOS is a waste of time for someone who just wants to write Markdown and push. Because GitHub Pages provides native Jekyll integration via GitHub Actions, your local machine only needs `git` and `nvim`.

You will write `.md` files, push to `main`, and the GitHub Actions runner will provision the Ruby environment, build the static HTML, and deploy it.

### 2. Initialize Minimal Jekyll Structure
Since we aren't using the CLI `jekyll new` (which requires local Ruby), manually scaffold the required directories and base files.

```bash
mkdir -p _posts .github/workflows
touch _config.yml Gemfile index.md CNAME .gitignore
```

### 4. `_config.yml` (Core Configuration)

> **Annotation:** Add a `plugins` key so `remote_theme` is picked up correctly.

```yaml
title: Aces Anderson
description: AI Engineering, LLMs, and Software Architecture.
url: "https://acesanderson.dev"

# Theme Configuration
remote_theme: "pages-themes/minimal@v0.2.0"

plugins:
  - jekyll-remote-theme

# Exclude raw files from the built HTML
exclude:
  - .github/
  - Gemfile
  - Gemfile.lock
  - README.md
  - pyproject.toml
  - uv.lock
  - src/
  - tests/

# Permalinks
permalink: /:year/:month/:day/:title/

# Markdown parser
markdown: kramdown
kramdown:
  input: GFM
  syntax_highlighter: rouge
```

### 5. `Gemfile`
Even though you aren't running Ruby locally, the GitHub Action reads this file to build the site with the correct dependencies.

```ruby
source "https://rubygems.org"

gem "github-pages", group: :jekyll_plugins
```

### 6. GitHub Actions Workflow
Add this to `.github/workflows/pages.yml`.

> **Annotation:** Before this workflow will trigger, go to repo Settings → Pages → Build and deployment → Source → **GitHub Actions**.

```yaml
name: Deploy Jekyll site to Pages

on:
  push:
    branches: ["main"]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Ruby
        uses: ruby/setup-ruby@v1
        with:
          ruby-version: '3.1'
          bundler-cache: true
      - name: Setup Pages
        uses: actions/configure-pages@v4
      - name: Build with Jekyll
        uses: actions/jekyll-build-pages@v1
        with:
          source: ./
          destination: ./_site
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### 7. Theme Selection
**Recommendation:** `pages-themes/minimal`
Officially supported GitHub Pages theme. Single-column, readable, natively supports syntax highlighting (Rouge) for Python/bash/YAML blocks, requires zero HTML/CSS customization. Already set in step 4 via `remote_theme`.

`index.md` — your homepage:
```markdown
---
layout: default
---

# Ace Sanderson

I build LLM pipelines, vector databases, and custom orchestration frameworks.

## Recent Posts
<ul>
  {% for post in site.posts %}
    <li>
      <a href="{{ post.url }}">{{ post.title }}</a> - {{ post.date | date: "%B %d, %Y" }}
    </li>
  {% endfor %}
</ul>
```

The `{% %}` blocks are Liquid templates — Jekyll processes them automatically. Paste as-is.

### 8. Custom Domain Setup (Porkbun & GitHub)
**In your repo:**
```bash
echo "acesanderson.dev" > CNAME
```

**In Porkbun DNS Management:**
Delete any default parked A/CNAME records, then add:

1. **A Records** — Host field: `@` (root domain)
   - `185.199.108.153`
   - `185.199.109.153`
   - `185.199.110.153`
   - `185.199.111.153`

2. **CNAME Record**
   - Host: `www`
   - Answer: `acesanderson.github.io.`

> **Annotation:** Porkbun uses `@` for the root domain in the host field, not blank.

**In GitHub repo settings:**
Settings → Pages → Custom domain → enter `acesanderson.dev` → Save. SSL certificate provisions automatically in 5–15 minutes.

### 9. Writing and Publishing Posts
